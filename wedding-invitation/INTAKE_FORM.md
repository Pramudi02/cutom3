# Wedding Invitation — Client Intake Form Guide

This document lists every piece of information needed from a couple to build their web invitation. Use it as the blueprint when creating a Google Form. Fields marked **★ Required** must be filled — everything else is optional but recommended.

---

## SECTION 1 — Bride & Groom Details

| #   | Question                                                | Type       | Notes                                       |
| --- | ------------------------------------------------------- | ---------- | ------------------------------------------- |
| 1   | **★ Bride's first name**                                | Short text | Used everywhere (hero, envelope, reveal)    |
| 2   | Bride's last name                                       | Short text | For formal/full name display                |
| 3   | **★ Groom's first name**                                | Short text | Used everywhere                             |
| 4   | Groom's last name                                       | Short text | For formal/full name display                |
| 5   | **★ Display names** (as it should appear on the invite) | Short text | e.g. `Kasun & Nimesha` or `Nimesha & Kasun` |

## SECTION 9 — Envelope Customisation

_(Only fill if a wax-seal envelope style was chosen in Section 8)_

| #   | Question                               | Type                                                       | Notes                                                                                                                                     |
| --- | -------------------------------------- | ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| 64  | Text above the couple name on envelope | Short text                                                 | e.g. `A Wedding Invitation`                                                                                                               |
| 65  | Button/seal text on envelope           | Short text                                                 | e.g. `Open Invitation`                                                                                                                    |
| 66  | Wax seal image                         | File upload (**PNG or WebP only, transparent background**) | Custom monogram, crest, or floral motif. Square, 500×500 px or larger. **JPEG not supported** — JPEG cannot have transparent backgrounds. |

> **Image guide for the wax seal:**
>
> - ✅ Good formats: **PNG**, **WebP**
> - ✅ The image should have a **transparent background** (checkerboard pattern when viewed in image editor)
> - ✅ Recommended size: **500×500 px or larger**, square crop
> - ❌ Do not use JPEG/JPG — it does not support transparency
> - ❌ Avoid white or coloured backgrounds (they will show as a solid block over the envelope)

## Summary: Minimum Required to Go Live

If a couple provides only the bare minimum, the invitation will still render correctly:

| Field                         | Value needed |
| ----------------------------- | ------------ |
| Bride first name              | ✅           |
| Groom first name              | ✅           |
| Display names                 | ✅           |
| Wedding date + time           | ✅           |
| Display date string           | ✅           |
| Invitation text               | ✅           |
| Story (1 paragraph)           | ✅           |
| Ceremony venue name + address | ✅           |
| WhatsApp RSVP number          | ✅           |
| RSVP deadline                 | ✅           |
| Hero photo                    | ✅           |
| Colour theme (preset)         | ✅           |
| Hero layout choice            | ✅           |
| Opening reveal style          | ✅           |
| Page title + description      | ✅           |
| URL slug                      | ✅           |

Everything else (gallery, music, timeline, custom colours, envelope customisation, couple reveal photos, section backgrounds, social OG image) enriches the experience but is truly optional.

## Google Form Structure Recommendation

Organise the form into these **sections/pages**:

1. **Page 1 — Couple Details** (Q1–6)
2. **Page 2 — Wedding Date & Time** (Q7–9)
3. **Page 3 — Story & Invitation Text** (Q10–12)
4. **Page 4 — Ceremony & Events** (Q13–22)
5. **Page 5 — Day Timeline** (Q23–26 × 6)
6. **Page 6 — RSVP** (Q27–29)
7. **Page 7 — Photos & Music** (Q30–58)
8. **Page 8 — Design & Style** (Q59–63)
9. **Page 9 — Envelope Details** (Q64–70, shown conditionally if envelope style chosen)
10. **Page 10 — SEO & Slug** (Q71–74)

> **Tip:** In Google Forms, use the "Go to section based on answer" feature on Q59 (reveal style) to show Page 9 only when an envelope-type is selected.
